      # flex布局使用在overflow: auto容器内时，溢出部分被修剪的解决办法？

      前面自己写的时候也发现了这个问题，解决了一会儿，百度flex overflow在segmentfault上也看到了别人的提问 我想要实现的一种样式是，父容器固定宽度，overflow-x: auto, 子容器宽度不定，如果小于父容器则局中，大于父容器则可以滚动。 对于display: flex的容器来说，它的宽度不会超出父容器的宽度，但是强行放入

      # 示例代码：



















      <div class="container">
    <div class="child"></div>
</div>
// 样式
.container {
    background-color: red;
    width: 1000px;
    height: 100px;
    display: flex;
    justify-content: center;
    overflow-x: auto;
}
.child {
    background-color: black;
    width: 200px;
    height: 100px;
}

      # 对于上图代码，样式没有问题，效果如下图

      然后修改下代码，将子容器的宽度调成1200px，效果如下图: 首先是滚动效果overflow-x: auto无效，然后是子容器的最大宽度就是100%，只有子容器的子容器撑开后才能有部分滚动效果，但会出现前面链接中的修剪问题。

      # 解决办法

      在父容器外再套一层div。

      # 代码：

























      <div class="outer">
    <div class="container">
        <div class="inner"></div>
    </div>
</div>
// 样式
.outer {
    display: flex;
    justify-content: center;
}
.container {
    background-color: red;
    width: 1000px;
    height: 100px;
    display: flex;
    justify-content: flex-start;
    overflow-x: auto;
}
.child {
    background-color: black;
    width: 200px;
    height: 100px;
}

      使用外层css, justify-content: center来让container居中，container过小则会成功居中，container最大则会百分百。container样式justify-content: flex-start让子容器在能滚动的时候不会发生裁剪。
